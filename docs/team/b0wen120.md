## Overview

### Enhancements & Architecture Implemented

**Core Architecture:** Designed and created the `UI` Component and Main Execution Loop
- **What it does:** Centralizes all CLI input and output operations and establishes the main application lifecycle loop.
- **Justification & Depth:** To prevent potential memory leaks and resource exhaustion from redundant stream creation, I engineered the `UI` class to encapsulate and maintain a single `Scanner` instance state. I also built the `ExpensiveLeh.java` startup sequence to orchestrate the welcome persona and route raw input to the `Parser`.

**New feature:** Data Visualization (`RankCommand`)
- **What it does:** Provides an advanced data visualization tool that aggregates, sorts, and displays complex data structures (expenses and loans) as formatted ASCII bar charts.
- **Justification & Depth:** This feature required cross-manager integration to seamlessly query both `ExpenseManager` (ranking by category) and `LoanManager` (ranking by person). It dynamically calculates the proportional length of each ASCII bar relative to the highest monetary value in the sorted list, ensuring accurate visual representation regardless of the data scale.

### Bug Fixes & Refactoring

**Parser Refactoring & UX Improvements:**
- **Strict Parsing & Order Enforcement:** Engineered a state machine (`lastSeenFlagOrder`) within the parsing loops to strictly enforce command flag sequences (e.g., ensuring `c/` precedes `n/`). Removed ambiguous defaults to force explicit user intent (e.g., requiring `edit expense` vs `edit loan`).
- **Solved the `add` bug:** Identified and fixed a critical logic gap where empty arguments bypassed validation and threw confusing system errors.
- **Enhanced the `edit` logic:** Refactored `parseEditCommand` to introduce Quality of Life (QoL) shorthand. Users can now use explicit commands (`edit loan 1`, `edit expense 1`).
- **Hardened Exception Handling:** Fixed "swallowed" exceptions by ensuring custom `ExpensiveLehException` messages properly bubbled up to the user, while keeping generic `Exception` catch blocks as safety nets.
- **Command Redirection:** Acted as a QA tester to identify UX friction points, successfully updating the parser to accept both `exit` and `bye`.
